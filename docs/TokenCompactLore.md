# 令牌密契 · Token Compact Lore & UX Spec

> 宠物成长系统的世界观、双层命名、途径分支、品质/颜色规范与数值目标。  
> **实现计划见** [`TokenCompactPlan.md`](./TokenCompactPlan.md)。  
> **像素表现仍遵循** [`PixelPetArtBible.md`](./PixelPetArtBible.md) / [`PixelPetRoadmap.md`](./PixelPetRoadmap.md)。

---

## 1. 定位

### 一句话

> 在 prompt 与 completion 的缝隙里，栖息着被 **「令牌密契」** 召唤的灵体 **Tokcat**。  
> 玩家消耗的 token 是它吞下的 **回响残片**；它并非简单长大，而是沿三条 **本源途径** 推进 **序列**，显化不同 **密契相**。

### 产品原则

| 原则 | 说明 |
|------|------|
| 本地优先 | 成长 / 背包 / 掉落均本地；默认离线 |
| Token 养成 | 复用现有适配器与 `PetEngine` 脉冲 |
| 双层表达 | **明文主显 + 密契副显**；怪名不得单独承担信息 |
| 途径分支 | 三基础属性对应三途径；门控解锁内容与器物 |
| 轻诡秘结构 | 借「途径 / 序列 / 扮演 / 晋阶」结构，不碰版权名物 |

### 调性

轻奇幻桌宠 + 工程师梗。神秘感用在命名与解锁文案，不做黑暗向重度叙事。

## 2. 双层命名（强制）

任何密契术语出场时，**同屏必须挂明文锚点**（数字、俗名、颜色、星级至少一种）。

| 系统概念 | 内部 / 兼容字段 | 明文（主显，新人） | 密契（副显） |
|----------|-----------------|--------------------|--------------|
| 等级 | `PetState.level` | **Lv.N** | 序列 S · 称号 |
| 经验 | `PetState.xp` | 经验 / 升级进度 | 回响 / 消化进度 |
| 日 XP 上限 | `dailyXPSoftCap` | 每日经验软上限 | 日消化阈值 |
| 智力 | `stats.intelligence` | **聪明** | 智识 |
| 体力 | `stats.vitality` | **稳定** | 存续 |
| 精力 | `stats.energy` | **手感** | 闪流 |
| 升级 | level up | **升级** | 晋阶 |
| 掉落 | loot | **掉落 / 获得** | 余响凝结 |
| 装备 | equipment | **装备** | 密契器物 |
| 皮肤 | skin | **皮肤** | 密契相 / 外相 |
| 道具 | prop | **道具** | 残片 / 戏物 |
| 成就 | achievement | **成就** | 密契证印 |
| 阶段外观 | `PetStage` 演进 | 新手→入门→… | 显化阶 |
| 菜单栏图标 | menu bar cat | 菜单栏猫 | 信标相 |

### 文案优先级（扫读顺序）

1. 数字等级 `Lv.n`
2. 品质颜色 + 明文品质
3. 三属性色条与整数
4. 红/绿需求勾
5. 途径色点 / 图标
6. 密契称号与 lore（装饰层）

### 显示模板

**档案主卡**

```text
Lv.12
序列 7 · 解读者          ← 副行，可弱化
聪明 14 · 稳定 9 · 手感 11
经验 128 / 210  (62%)
下一解锁：聪明线眼镜 · 还需 聪明 0（已满足）/ Lv.10
```

**升级 Toast**

```text
升级！ Lv.11 → Lv.12
序列 8 → 7 · 解读者
```

**掉落 Toast**

```text
获得：旁注单片镜
稀有 · 眼镜
```

**装备需求（永远明文）**

```text
需要 Lv.10 · 聪明 ≥ 8
效果：稀有掉落权重 +15%
```

### 模式策略（本阶段）

**不做复杂开关。** 全 UI 统一：

- **主显**：Lv、俗名属性、普通→传说、帽子/眼镜…、±% 效果  
- **副显**：序列称号、智识/存续/闪流、凡响→源响、一句设定  

后续若需要，再加「密契浓度」设置（简明 / 标准 / 浓密契）。

## 3. 三条本源途径

三条可 **并行修持**；`max(聪明,稳定,手感)` 定 **主途**，次高为 **辅途**（差值过小显示「双生」）。

| 途径 ID | 密契名 | 明文标签 | 属性 | 扮演（Acting） | 主题色 | 气质 |
|---------|--------|----------|------|----------------|--------|------|
| `reader` | 解读者 | **聪明线** | 智识 / 聪明 | 高单价、推理/重模型 | 墨青 × 金箔 `#1E3A5F` / `#D4AF37` | 镜片、符文、书页 |
| `warden` | 守夜人 | **稳定线** | 存续 / 稳定 | 每日归来、streak、细水长流 | 炉火橙 × 烟灰 `#E89B4B` / `#6B7280` | 围巾、灯火、暖边 |
| `flash` | 疾响者 | **手感线** | 闪流 / 手感 | 低延迟、高速吞吐 | 电光青 × 午夜 `#22D3EE` / `#0F172A` | 冷帽、键帽、电弧 |

### 途径授衔（称号随「途径进度」升，不等于全局 Lv）

全局用 **Lv** 做时间轴；途径称号由 **Lv 门槛 + 属性门槛** 共同点亮。  
下列为 **聪明线（解读者）** 示例，稳定线 / 手感线见同构表。

| 途径阶 | 称号（聪明线） | 建议门槛（明文） |
|--------|----------------|------------------|
| 0 | 拾句者 | 默认 |
| 1 | 旁注生 | Lv.5 · 聪明 ≥ 4 |
| 2 | 解读者 | Lv.10 · 聪明 ≥ 8 |
| 3 | 校注官 | Lv.15 · 聪明 ≥ 12 |
| 4 | 推演师 | Lv.20 · 聪明 ≥ 18 |
| 5 | 典藏使 | Lv.30 · 聪明 ≥ 24 |
| 6 | 沉默文库 | Lv.40 · 聪明 ≥ 30 |
| 7 | 因果校对 | Lv.50 · 聪明 ≥ 40 |
| 8 | 全知之隙 | Lv.60 · 聪明 ≥ 50 |
| 9 | 令牌解经者 | Lv.75 · 聪明 ≥ 55 |

**稳定线称号**：未归猫 → … → 守夜人 → … → 永续守门猫  
**手感线称号**：静电崽 → … → 疾响者 → … → 瞬答之猫  

实现可用 `PathwayTitleTables` 静态表；UI 默认显示「当前最高已点亮称号」。

### 启程 / 定契 / 高座（内容门控）

| 阶段 | 明文 | 条件（示例） | 解锁 |
|------|------|--------------|------|
| 启程 | 解锁该成长线 | Lv≥10 且对应属性≥8 | 专属掉落池、该线低阶装备可装 |
| 定契 | 进阶线 | Lv≥20 且属性≥18（+扮演条件可选） | 稀有装备、皮肤偏置 |
| 高座 | 高阶线 | Lv≥35 且属性≥30 | 史诗池、高级冠徽 |
| 均衡隐藏 | 全能线 | 三属性均≥22 | 特殊光环 / 证印 |

未启程：仍可升 **全局 Lv**，但专属池不进、高阶器物灰锁。

## 4. 序列与显化阶（替代「幼猫/成猫/老猫」主叙事）

内部继续存 `level: Int`（1…）。对外：

| 显化阶 ID | 明文阶段 | 密契显化 | 约 Lv | 约序列显示 | 阶段色 |
|-----------|----------|----------|-------|------------|--------|
| `spark` | 新手 | 初契·显影 | 1–9 | 序列 9–8 | 灰白 |
| `initiate` | 入门 | 入途·启灵 | 10–19 | 序列 7–6 | 绿 |
| `formed` | 进阶 | 定契·成形 | 20–34 | 序列 5–4 | 蓝 |
| `sanctum` | 高阶 | 高座·显圣 | 35–54 | 序列 3–2 | 紫 |
| `nearDivine` | 巅峰 | 近神·权柄 | 55–74 | 序列 1 | 橙金 |
| `sovereign` | 满阶 | 座上·令主 | 75+ | 序列 0 | 金闪 |

**序列数字对外可选显示**：`sequenceLabel(level)` → 9…0 映射。  
**新人只看 Lv + 明文阶段色即可。**

> 旧 `PetStage.kitten/adult/elder`：迁移期映射到 `spark–initiate` / `formed` / `sanctum+`，文案层废弃「老猫」。

## 6. 装备槽与权能

| 槽 | 明文 | 密契 | 主倾向 | 权能方向 |
|----|------|------|--------|----------|
| `head` | 帽子 | 冠徽 | 全途 / 信标 | 菜单栏帽子；小幅经验 |
| `face` | 眼镜 | 目镜 | 聪明线 | 稀有权重↑ |
| `back` | 披风/围巾 | 披引 | 稳定线 | 饥饿衰减↓、日经验上限↑ |
| `held` | 手持 | 契物 | 手感线 | 掉落率↑、工作演出 |
| `aura` | 光环 | 灵晕 | 高阶 | 全局小加成 |

### 效果字段（数值）

```text
xpMultiplier          // 1.03 = +3% 经验
dropChanceBonus       // +0.01 absolute
rarityWeightBias      // 稀有档权重倍率
hungerDecayMultiplier // 0.9 = 饿得慢 10%
dailyXPSoftCapBonus   // +15
menuBarHatID          // 菜单栏叠加
```

### 叠加硬顶

| 类型 | Cap |
|------|-----|
| 经验加成合计 | ≤ +20% |
| 掉落率加成合计 | ≤ +8% absolute |
| 日经验上限加成 | ≤ +40 |
| 稀有权重偏置 | 实现时再定软顶 |

### 门槛失败策略

- **外观**：可保留已装备外观（友好）  
- **数值权能**：不满足 `minLevel` / 属性需求时 **效果关闭**，UI 标「权能休眠」  
- 首次不满足时轻提示一次

## 7. 模型 → 回响品质与扮演

沿用 `NutritionTier`，文案层：

| Tier | 明文 | 密契 | 主滋养 |
|------|------|------|--------|
| premium | 高价模型 | 高位残篇 | 聪明 + 较高经验 |
| standard | 常规模型 | 常响 | 均衡 |
| economy | 低价/本地 | 粗制残片 | 稳定为主、经验偏低 |

扩展（计划内）：

```text
ModelProfile {
  nutrition
  tempo: fast | normal | slow     // 延迟/手感
  pathwayAffinity: reader|warden|flash?
  growthBias: { int, vit, en }
}
```

| 行为 | 主属性 | 明文提示 |
|------|--------|----------|
| 高单价 | 聪明 | 想变聪明：多用贵模型 |
| 每天都写 | 稳定 | 想更稳定：保持连续活跃 |
| 低延迟 | 手感 | 手感来自更快的响应 |

## 8. 掉落（余响）规则目标

| 参数 | 目标默认 | 明文说明 |
|------|----------|----------|
| feedBaseChance | 0.05 | 每次有效喂食 5% |
| firstFeedBonus | 0.10 | 每日首次 +10% |
| pityThreshold | 18 | 连续落空补偿 |
| dailyCap | 4 | 每日最多 4 次掉落 |
| minTokensForRoll | 2000 | 过碎批次不 roll |

### 池过滤

```text
eligible =
  通用道具
  ∪ 已启程途径装备/道具
  ∪ 序列(Lv) 允许的稀有带
  ∪ 满足 minLevel 的条目
```

### 稀有度权重随 Lv 上移

| 玩家 Lv | 普通 | 优秀 | 稀有 | 史诗 | 传说 |
|---------|------|------|------|------|------|
| 1–9 | 75 | 22 | 3 | 0 | 0 |
| 10–24 | 55 | 30 | 13 | 2 | 0 |
| 25–49 | 40 | 32 | 20 | 7 | 1 |
| 50+ | 30 | 30 | 25 | 12 | 3 |

升级礼：保底 ≤优秀；每 5/10 级额外偏稀有 roll。

## 9. 成长数值目标（Balance Snapshot v2）

> 具体常量落在 `GrowthBalance`；此处为设计锚点。

### 节奏（日 softcap 吃满的中度用户）

| 里程碑 | 目标 |
|--------|------|
| Lv 5 | 2–4 天 |
| Lv 10 | ~2 周 |
| Lv 25 | ~1–1.5 月 |
| Lv 50 | 长线数月 |
| Lv 75+ | 满阶追求 |

### 建议公式

```text
xpToNext(L)      = 150 + 55 * L^1.48
xpPerToken       = 1 / 1800
tierXP           = economy 0.65 / standard 1.0 / premium 1.35
dailyXPSoftCap   = 220
overflowRate     = 0.12

# 属性（相对旧版大幅降速）
int: premium * xpPerToken * 0.35, standard * 0.08, economy * 0.02
vit: continuity * 0.08 + token drip * 0.04
en:  latencyMood * 0.04
statSoftCap      = 100, after gain *= 0.15
```

### 旧版问题（对照）

| 项 | v1 现状 | 问题 |
|----|---------|------|
| XP 曲线 | `80+45*L^1.35` | 早期过快 |
| xpPerToken | 1/650 | 产出过高 |
| daily softcap | 420 | 日升级过多 |
| 属性增长 | INT×2 等 | 成就阈值秒破 |
| 装备 | 纯外观 | 无 build |
| 阶段名 | 幼/成/老猫 | 缺 IP |

## 12. 文案组件约定（给实现）

```text
LevelLabel        = "Lv.\(n)" + optional " · 序列\(s) · \(title)"
StatPlain         = 聪明 | 稳定 | 手感
StatLore          = 智识 | 存续 | 闪流

PathwayStatus     = "聪明线 · 已解锁" | "未解锁 · 还需…"
```
